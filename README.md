# AI Claim Assistant

## Introduction
AI Claim Assistant is an intelligent insurance claim processing system designed to automate medical insurance claim verification and reporting.

The system combines:
- OCR (Optical Character Recognition)
- AI-based text extraction
- Rule-based validation
- PDF report generation
- Email automation

The main objective of the project is to reduce manual effort, improve claim processing speed, minimize fraud, and provide accurate insurance claim analysis.

---

# Objectives

- Automate insurance claim processing
- Extract data from medical bills automatically
- Validate policy and patient information
- Calculate insurance claim amount based on policy type
- Generate PDF reports automatically
- Send automated email notifications
- Reduce manual verification effort
- Improve operational efficiency

# Working of the System

## Step 1: User Login
The user enters username and password to access the system.

## Step 2: Policy Verification
The system verifies whether the policy number exists in the database.

## Step 3: Bill Upload
The user uploads a medical bill image.

## Step 4: Temporary File Handling
The uploaded image is stored temporarily because OCR requires a file path.

## Step 5: OCR Processing
PaddleOCR extracts text from the uploaded bill.

## Step 6: AI Processing
The extracted text is sent to Ollama AI models for structured information extraction.

## Step 7: Validation
The system verifies:
- Patient name
- Doctor details
- Hospital details
- Policy information

## Step 8: Claim Calculation

### Claim Percentage Logic
- Basic Policy → 60%
- Premium Policy → 80%
- Ultra Policy → 100%

## Step 9: PDF Generation
A detailed insurance report is generated automatically.

## Step 10: Email Notification
The system sends approval/rejection emails automatically.

---

# Claim Scoring Logic

The system uses a score-based validation method.

| Validation Parameter | Score |
|---------------------|-------|
| Patient Name | 20 |
| Doctor Name | 20 |
| Doctor Validation | 20 |
| Hospital Name | 20 |

## Approval Criteria
If total score is greater than or equal to 80, the claim is approved.

# Technical Explanation

## OCR Processing
The system uses PaddleOCR to convert medical bill images into machine-readable text. OCR processing helps automate manual data entry.

## AI Data Extraction
The extracted OCR text is sent to Ollama AI models such as Mistral and Phi3. The AI model analyzes the text and returns structured JSON data.

## Rule-Based Validation
The system validates:
- Patient name
- Policy details
- Doctor information
- Hospital details

The validation process helps reduce fraudulent claims.

## Claim Decision Engine
The decision engine calculates a score and determines whether the claim should be approved or rejected.
